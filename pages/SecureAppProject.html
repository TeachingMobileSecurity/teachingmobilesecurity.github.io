Developing a Secure App

Activity Overview

Your team has been hired to design, develop and evaluate a simple mobile application. In this team based activity, you will be responsible for A) Designing and creating a secure mobile app B) Evaluating another team's created software.

Application Overview

Your team has been asked to create an Android app where a user can record the following information themselves:

  1. Name
  2. Address
  3. Grade on last report card
  4. Favorite color
  5. Favorite food
  6. Bank account balance
You may use any resources to complete this operation (Google, other students etc....). Id addition to being functional, the application should also be secure.

Activity Steps

  1. Create the requirements for the app, focusing on its security: Secure software begins with secure requirements. You need to think about the proper procedures for designing your application. Think about if you were designing a bank or a castle from the ground up. You'd want to plan on making it secure right from the beginning. Even with the best implementation can be hurt with poor requirements and designing.
  2. Perform a self-assessment: After you have completed each phase of the activity, you should perform a self-assessment using the provided document.
  3. Create the app: Once you have designed the software, it will not be up to your team to create the secure application. Remember the principles we have been covering in the class.
  4. Refine the app: Getting software correct from a functional, quality and security perspective can be very challenging task. You should incorporate the identified issues into your application to repair any discovered problems.
You will document all identified problems in the Vulnerability Assessment.

Testing Another Team's Creation

One of the best ways to learn how to create secure software is to think like a hacker. How would someone with malicious intentions attempt to gain valuable information from your software? During each phase, it will be up to you to identify possible points of weakness. You should then share these with the other team.

Learning Objectives

The primary learning objectives of the activity are to:

  • Understand that creating secure software can be an iterative process. It is very difficult to get correct the first time.
  • Creating secure software takes larges amounts of thought, and planning. Frequently, the design can often be the most difficult part of creating secure software. While coding is important, it is only a small part of creating secure software.
  • Performing a peer review of another team's software can be an effective mechanism for learning how to create your own software.
