Learning from Existing Mobile Vulnerabilities

Activity Overview

Understanding previous vulnerabilities is a great way to learn more about cybersecurity. In this activity, you will read about a previous mobile vulnerability and report on the following:

  • What was the vulnerability? (one paragraph)
  • What caused it (from a process perspective)
  • What caused it (from a technical perspective)
  • What was the impact of the vulnerability?
  • Who was impacted?
  • How would you recommend for preventing it?
  • What type of vulnerability was it?
  • Does it have a CWE/CVE (provide link)

Resources for Historical Information

The web is full of great resources for learning about previous vulnerabilities. A few are listed below:

  • Common Weakness Enumeration: CWE is a community-developed list of common software security weaknesses. It serves as a common language, a measuring stick for software security tools, and as a baseline for weakness identification, mitigation, and prevention efforts.
  • Common Vulnerabilities and Exposures: CVE is a list of entries—each containing an identification number, a description, and at least one public reference—for publicly known cybersecurity vulnerabilities.
